The Missing Link in Your Production
In today's competitive landscape, companies seek game changing capabilities like canary releases, testing in production and practical observability for their Kubernetes based microservices applications. These goals are often out of reach due to an established fallacy: Deployments and releases are being conflated into a single phase by existing CI/CD tools and pipelines.
At Bevyx, we've recognized that release is an important independent phase and should be treated that way. We call it Continuous Release.
Learn more about Continuous Release

Meet Remesh

Remesh is an open source solution that delivers Continuous Release capabilities for Kubernetes. By building on top of Istio Service Mesh technology, it provides a simple and practical solution to support multiple coexisting releases in your microservices environment.

A/B Testing

Remesh makes A/B testing an easy task even for complex Kubernetes based applications.


Testing in Production

Remesh ensures that running tests in production and testing new releases with  real traffic is safe and controlled.


Segment Targeting

Remesh allows you to slice your audience into custom addressable segments.
